经过测试，我得到了总体的混淆矩阵，发现了一些问题：
1. Class 0 / Class 1 / Class 2 之间混淆明显；
2. Class 3 / Class 4 / Class 5 之间混淆明显；
3. Class 10 / Class 11 之间混淆明显；
4. Class 12 和 Class 13 的漏检率仍较大；
5. Class 9 的误报率较大；

现在我想先解决问题1、问题2和问题3，因为我感觉它们都是同一问题：对同类信号不同调制方式的情况检测混淆严重。

目前我自己只反思出一条原因：STFT处理后本来得到的二维数据的每个数据点都是一个复数，但是我直接取模并保存为png，即三通道均为模的灰度图像，从而丢弃了相位信息，可能导致 QPSK / 16QAM / 64QAM 之间的混淆严重。

目前我自己也只想出来一条可能的改进方案（包含两种待测试哪个好的情况）：对数据进行STFT处理后，得到一个每个数据点都是一个复数的二维数据。
第一种情况：对每个点取模得到一个通道的图像，对每个点取实部得到一个通道的图像，对每个点取虚部得到一个通道的图像，将这三个图像封装为RGB三通道图像保存；
第二种情况：对每个点取模得到一个通道的图像，对每个点求正弦值得到一个通道的图像，对每个点求余弦值得到一个通道的图像，将这三个图像封装为RGB三通道图像保存；
我感觉这样处理可以引入相位，从而更好地区分同类信号的不同调制方式；

（而且我还感觉，如果使用第二种情况，应该对模通道使用maxpool，对正弦/余弦通道使用avgpool，这样即能保证小信号不丢失，模特征明显，又能在相位上确保连续性。但是由于这个改进需要触及模型内部结构，你可以评价一下合理性，但是咱们暂且不做修改）
